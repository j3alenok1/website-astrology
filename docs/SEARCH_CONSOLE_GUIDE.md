# Google Search Console: полный гид

Пошаговые инструкции по работе с [Search Console](https://search.google.com/search-console) для **astrobyndauzh.com**.

---

## 1. Дополнительные страницы (/astrolog-online, /astrolog-moscow) — что это и зачем

### Идея

Люди ищут по-разному:
- «астролог алматы» → уже есть /astrolog-almaty
- «астролог онлайн» → **нет** страницы
- «астролог москва» → **нет** страницы
- «астролог россия» → **нет** страницы

Каждый новый запрос = отдельный лендинг. Больше страниц под разные запросы → больше показов в выдаче.

### Как это работает

| Страница | Запрос | Кому подходит |
|----------|--------|----------------|
| `/astrolog-almaty` | астролог алматы, астролог в алматы | Жители Алматы |
| `/astrolog-online` | астролог онлайн, консультация астролога онлайн | Все, кто ищет удалённо |
| `/astrolog-moscow` | астролог москва | Жители Москвы (работаете онлайн) |
| `/astrolog-russia` | астролог россия | Вся РФ |
| `/astrolog-kazakhstan` | астролог казахстан | Вся страна |

**✅ Реализовано** — все эти страницы созданы и добавлены в sitemap.

### Важно

Офиса в Москве нет — это нормально. Текст: «Работаю онлайн с клиентами из Москвы, Алматы, России, Казахстана. Консультация по Zoom или WhatsApp.» Честно и без обмана.

### Результат

Каждая такая страница может дать 10–100+ показов в месяц по своему запросу. Суммарно — рост охвата.

## 2. Как проверить индексацию

### Шаг 1: Открыть Search Console

1. Зайти на https://search.google.com/search-console
2. Выбрать ресурс **astrobyndauzh.com**

### Шаг 2: Проверить количество страниц в индексе

1. Меню слева → **«Индексирование»** (Indexing)
2. Подменю → **«Страницы»** (Pages)
3. Смотреть блок **«Проиндексировано»** (Indexed) — сколько URL в индексе Google

**Норма:** минимум 3–5 страниц (главная, astrolog-almaty, relationshipastrology, privacy и т.д.)

**Проблема:** «0 проиндексировано» или очень мало — сайт не виден в поиске.

### Шаг 3: Проверка отдельных URL

1. Меню слева → **«Проверка URL»** (URL Inspection)
2. Ввести адрес, например: `https://astrobyndauzh.com/astrolog-almaty`
3. Нажать Enter
4. Смотреть статус: «URL есть в Google» или «URL отсутствует в Google»

Если **«URL отсутствует»** → нажать **«Запросить индексирование»** (Request indexing). Google добавит страницу в очередь на обход.

### Шаг 4: Проверить sitemap

1. Меню → **«Индексирование»** → **«Файлы Sitemap»** (Sitemaps)
2. Должен быть добавлен: `https://astrobyndauzh.com/sitemap.xml`
3. Если нет — нажать «Добавить новый файл sitemap», ввести `sitemap.xml`
4. В sitemap должны быть все важные страницы (главная, astrolog-almaty, astrolog-online, astrolog-moscow, astrolog-russia, astrolog-kazakhstan и др.). Тогда в «Проверке URL» будет указано «Файлы Sitemap» как источник обнаружения.

### Шаг 5: Исключённые страницы

1. **«Индексирование»** → **«Страницы»**
2. Посмотреть блок **«Исключено»** (Excluded) — почему страницы не в индексе
3. Частые причины: «Обнаружено — в индексе не отображается», «Страница помечена как "Без индекса"»

## 3. Ссылочная масса — что это и как увеличить

### Что такое ссылочная масса

**Обратные ссылки (backlinks)** — это ссылки с других сайтов на ваш.  
Google воспринимает их как «рекомендации»: чем больше качественных ссылок, тем выше доверие и позиции.

**Ссылочная масса** = совокупность всех обратных ссылок на сайт (количество и качество).

### Как это помогает

- Больше ссылок с хороших сайтов → выше позиции
- Ссылки с тематических площадок (астрология, психология) — ценнее
- Ссылки с 2GIS, каталогов, статей — приносят трафик и вес

### Как увеличить (по приоритету)

| Действие | Где | Сложность |
|----------|-----|------------|
| Добавить в каталоги | Sxodim, Eshko, Kido, Zoon, Flamp | Легко |
| Google Business / Яндекс.Бизнес | business.google.com, yandex.ru/business | Легко |
| Ссылка в соцсетях | Instagram, VK, Telegram — в описании профиля | Легко |
| Отзыв с ссылкой | otzovik.com, irecommend.ru | Легко |
| Гостевая статья | Блог психолога, коуча | Средне |
| Обмен ссылками | Партнёр в нише | Средне |
| Платный пост | Астроканал в Telegram | Платно |

## 4. Полный список разделов Search Console

### Обзор (Overview)

- Общая картина: клики, показы, CTR, позиция
- Быстрые уведомления об ошибках

### Эффективность (Performance) → «Статистика»

- **Клики** — переходы из поиска
- **Показы** — сколько раз сайт был в выдаче
- **CTR** — клики / показы
- **Позиция** — средняя позиция в результатах

Вкладки:
- **ЗАПРОСЫ** — по каким фразам показывают и кликают
- **СТРАНИЦЫ** — какие URL дают трафик
- **СТРАНЫ** — география
- **УСТРОЙСТВА** — мобильные / десктоп
- **ВИД В ПОИСКЕ** — обычный поиск, картинки и т.д.
- **ДНИ** — динамика по дням

**Что делать:** смотреть «ЗАПРОСЫ» и «СТРАНИЦЫ», находить запросы с показами, но без кликов — улучшать заголовки и сниппеты.

### Индексирование (Indexing)

- **Страницы** — сколько URL в индексе, сколько исключено
- **Файлы Sitemap** — добавлен ли sitemap.xml
- **Удаления** — запросы на удаление URL (если были)

**Что делать:** убедиться, что проиндексировано достаточно страниц, sitemap добавлен и без ошибок.

### Проверка URL (URL Inspection)

- Проверка отдельных страниц
- Запрос индексирования

**Что делать:** для новых и важных страниц проверять статус и при необходимости запрашивать индексирование.

### Качество (Enhancements)

- **Core Web Vitals** — скорость и удобство
- **Мобильные устройства** — адаптивность
- **HTTPS** — защищённое соединение
- **Структурированные данные** — ошибки в разметке

**Что делать:** исправлять критичные ошибки, особенно по структурированным данным.

### Ссылки (Links)

- **Внешние** — кто ссылается на ваш сайт
- **Внутренние** — ссылки между страницами сайта

**Что делать:** следить за ростом внешних ссылок и проверять, что нет битых внутренних.

### Настройки (Settings)

- Владельцы и пользователи
- Переезд домена (если планируется)
- Параметры URL (если используются UTM и т.п.)
